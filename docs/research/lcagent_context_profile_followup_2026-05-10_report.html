Summary

Large-context mode is worth keeping. It eliminated overlapping read ranges in every run attempted here, while keeping the model informed enough to complete the task without forced synthesis in the completed rows.

The central harness change under test was not a new tool or a cheaper heuristic. It was a context-retention profile: keep the default loop conservative, but allow large-window routes to carry more raw tool evidence before transcript compaction. This directly targets the repeated-read pattern observed in earlier LCAgent benchmark sessions.

GPT-5.5 with low reasoning effort produced the strongest final answer in this follow-up, with a working score of 8.6. GPT-5.5 with no reasoning effort was cheaper and still strong, but less crisp. Kimi K2.6 remained the best lower-cost large-context route in this batch, while GLM 5.1 improved markedly versus its earlier low-effort result.

Comparison To Earlier Runs

Large context improved the harness behavior most clearly on read overlap. In an earlier generous-read run, DeepSeek used 40 reads, 7,620 read lines, and 2,275 overlapping lines, then still produced a wobbly final. With large context, it used 23 reads, 4,666 lines, zero overlap, no forced synthesis, and a lower estimated cost.

This suggests the repeated-read problem was partly harness-induced. Early compaction removed too much raw evidence from the live conversation, so models re-read central files. Keeping more evidence available is cleaner than making the model read fewer lines or adding brittle duplicate-read blocks.

The tradeoff is context volume. Large-context mode should stay opt-in for now, used when a model route has enough context window and cache economics to justify it.

Recommendation

Harness Defaults

  • Keep --context-profile balanced as the default.
  • Use --context-profile large for large-window benchmark lanes.
  • Track compaction count and forced synthesis in future benchmark tables.

Model Routes

  • Use GPT-5.5 low as the quality ceiling for this task, with cost caveats.
  • Keep Kimi K2.6 as the stronger lower-cost large-context candidate.
  • Rerun GLM on another task before dismissing it; large context improved it a lot.
